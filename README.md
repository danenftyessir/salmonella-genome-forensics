## Overview

SalmoTrace-BERT investigates whether genomic features can predict the **isolation source** of *Salmonella enterica* isolates (food, food-animal, environment, human) — a task known as *genomic source attribution*.

The pipeline implements five experiments (E1–E5) comparing different feature representations:

| Experiment | Features | Method |
|:---:|---|---|
| E1 | SNP pairwise distance | Hierarchical clustering (biological baseline) |
| E2 | Integer-encoded SNP matrix | Random Forest / SVM |
| E3 | DNABERT-2 mean-pooled embeddings | Random Forest / SVM |
| E4 | SNP + DNABERT concatenated | Random Forest / SVM |
| E5 *(optional)* | Tetranucleotide k-mer frequency | Random Forest / SVM |

Anti-leakage split: `StratifiedGroupKFold` on `snp_cluster` ensures isolates from the same genomic cluster never span train and test sets.

## Reproducibility

All sources of randomness are fixed via a single seed (`random_state: 42` in `config.yaml`). The same seed is applied to the Python hash, NumPy, scikit-learn splits, Random Forest, and PyTorch (for DNABERT inference).

### Setup

```bash
# 1. Clone
git clone <repo-url>
cd salmonella-genome-forensics

# 2. Create virtual environment
python -m venv .venv

# Windows
.venv\Scripts\activate
# Linux / macOS
source .venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# PyTorch CPU (if not auto-installed)
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

### Data download

1. Go to [NCBI Pathogen Detection](https://www.ncbi.nlm.nih.gov/pathogens/)
2. Search for *Salmonella enterica*, download metadata CSV → `data/raw/metadata/metadata_salmonella.csv`
3. Download FASTA genome files for each `assembly_accession` → `data/raw/genomes/<accession>.fna`
4. Download reference genome `GCF_000006945.2` → `data/raw/genomes/GCF_000006945.2.fna`

### Run

```bash
# Full pipeline (all stages, with checkpoint caching)
python main.py

# Force recompute everything from scratch
# Edit config.yaml: pipeline.force_recompute: true
python main.py

# Or run interactively via notebook
jupyter notebook notebooks/salmogen_trace_pipeline.ipynb
```

Set `FORCE_RECOMPUTE = False` in the notebook to load cached artifacts (fast demo mode).  
Set `FORCE_RECOMPUTE = True` to rerun all stages from scratch.

### Reproduce exact results

The following artifacts lock the experiment state:

| File | Purpose |
|---|---|
| `config.yaml` | All hyperparameters and paths |
| `artifacts/reports/manifest.json` | Dataset provenance, model config, best metrics |
| `artifacts/reports/metrics.json` | Per-mode metrics (E2/E3/E4) including `train_ids` / `test_ids` |
| `artifacts/reports/forensic_report.txt` | Per-isolate nearest-neighbor and source prediction |

To verify that a re-run reproduces the same train/test split, compare `metrics.json → all_modes → <mode> → train_ids` with the new run's output.

### Enable E5 k-mer baseline

```yaml
# config.yaml
kmer:
  enabled: true
  k: 4
```

## Configuration

All parameters live in `config.yaml`. Key sections:

```yaml
project:
  random_state: 42   # single seed for all random operations

ml:
  model: RandomForest
  target_col: isolation_source
  test_size: 0.2
  random_state: 42   # mirrors project.random_state
  n_estimators: 100

pipeline:
  force_recompute: false  # set true to ignore all cached artifacts
```

## Artifacts & Caching

Each pipeline stage writes a checkpoint artifact. On re-run, the stage is skipped if the artifact exists and `force_recompute: false`.

```
Stage 1 → data/interim/clean_metadata.parquet
Stage 2 → data/interim/sequence_quality.csv
Stage 3 → artifacts/matrices/snp_matrix.parquet
          artifacts/matrices/snp_matrix_encoded.parquet
          artifacts/matrices/distance_matrix.csv
Stage 4 → artifacts/embeddings/dnabert_embeddings.npz
Stage 5 → artifacts/models/rf_model.joblib
          artifacts/reports/metrics.json
          artifacts/reports/manifest.json
```

DNABERT embeddings (`.npz`) and trained models (`.joblib`) are excluded from git — regenerate them locally or share via a separate channel (e.g., Google Drive, Zenodo).

## Biological Context

Source attribution asks: *given a genome sequence, which environment did this isolate come from?*

The genomic basis: SNP distance reflects evolutionary divergence. Isolates with smaller SNP distances share a more recent common ancestor and are more likely to have originated from the same transmission chain or contamination source. This is the principle behind WGS-based outbreak investigation as used by NCBI Pathogen Detection.

> **Important limitation**: this project uses public NCBI data as a computational simulation. Results cannot be used to make direct claims about specific real-world contamination events.
